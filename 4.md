## **Operators in Python**
Python operators are the special symbols that perform operations on the values and variables

### **1. Assignment Operators**

Assignment operators are used to assign values to variables. The simplest one is `=` which assigns the value on the right to the variable on the left. There are also compound assignment operators that combine arithmetic operations with assignment.

#### **Common Assignment Operators:**
- `=`: Assigns value on the right to the variable on the left.
- `+=`: Adds right operand to the left operand and assigns the result to the left operand.
- `-=`: Subtracts the right operand from the left operand and assigns the result to the left operand.
- `*=`: Multiplies the left operand by the right operand and assigns the result to the left operand.
- `/=`: Divides the left operand by the right operand and assigns the result to the left operand.
- `%=`: Takes modulus of left operand by right operand and assigns the result to the left operand.

#### **Examples**:
```python
x = 5  # Assigns 5 to x
x += 3  # Equivalent to x = x + 3, now x is 8
x -= 2  # Equivalent to x = x - 2, now x is 6
x *= 4  # Equivalent to x = x * 4, now x is 24
x /= 6  # Equivalent to x = x / 6, now x is 4.0
```

#### **🎯 Interview Gotcha — `+=` on mutable vs immutable objects:**
```python
# With a list (mutable) - += modifies the SAME object in place
a = [1, 2, 3]
b = a
a += [4]
print(a, b)  # Output: [1, 2, 3, 4] [1, 2, 3, 4]  -> b changed too!

# With an int (immutable) - += creates a NEW object
x = 5
y = x
x += 1
print(x, y)  # Output: 6 5  -> y is unaffected
```
This trips people up in interviews: `+=` behaves differently depending on whether the object is **mutable** (list, dict, set) or **immutable** (int, float, str, tuple).

### **2. Comparison Operators**

Comparison operators are used to compare two values. They return either `True` or `False` depending on the condition.

#### **Common Comparison Operators**:
- `==`: Checks if two values are equal.
- `!=`: Checks if two values are not equal.
- `>`: Checks if the left operand is greater than the right operand.
- `<`: Checks if the left operand is less than the right operand.
- `>=`: Checks if the left operand is greater than or equal to the right operand.
- `<=`: Checks if the left operand is less than or equal to the right operand.

#### **Examples**:
```python
a = 10
b = 20

print(a == b)  # Output: False
print(a != b)  # Output: True
print(a > b)  # Output: False
print(a < b)  # Output: True
print(a >= 10)  # Output: True
print(b <= 25)  # Output: True
```

#### **🎯 Interview Gotcha — Chained comparisons:**
```python
x = 5
print(1 < x < 10)        # Output: True  -> same as (1 < x) and (x < 10)
print(10 < x < 1)        # Output: False

# Comparing across types
print(5 == 5.0)           # Output: True  -> value is compared, not type
print(5 == "5")            # Output: False -> different types, no implicit conversion
```

### **3. Logical Operators**

Logical operators are used to combine conditional statements. They evaluate expressions and return either `True` or `False`.

#### **Common Logical Operators**:
- `and`: Returns `True` if both conditions are true.
- `or`: Returns `True` if at least one condition is true.
- `not`: Reverses the logical state of its operand (True becomes False, and vice versa).

#### **Examples**:
```python
x = 5
y = 10
z = 15

# and operator
print(x > 0 and y > 5)  # Output: True (both conditions are True)

# or operator
print(x > 10 or z > 10)  # Output: True (one of the conditions is True)

# not operator
print(not(x > 10))  # Output: True (reverses False to True)
```

#### **🎯 Interview Gotcha — `and`/`or` return a VALUE, not just True/False (short-circuit evaluation):**
```python
print(5 and 10)     # Output: 10  -> both truthy, returns the LAST evaluated operand
print(0 and 10)     # Output: 0   -> first operand is falsy, so it's returned immediately

print(5 or 10)       # Output: 5   -> first operand is truthy, returned immediately (short-circuits)
print(0 or 10)       # Output: 10  -> first is falsy, so it evaluates and returns the second

# Common real-world use: default values
name = None
display_name = name or "Guest"
print(display_name)   # Output: "Guest"
```

### **4. Identity Operators**

Identity operators check whether two variables **refer to the same object in memory**, not whether their values are equal. This is one of the most commonly confused topics in interviews — people mix up `is` with `==`.

#### **Identity Operators**:
- `is`: Returns `True` if both variables point to the **same object** in memory.
- `is not`: Returns `True` if both variables point to **different objects** in memory.

#### **`is` vs `==` — the key difference:**
- `==` compares **values** (are the contents equal?).
- `is` compares **identity** (are they literally the same object in memory?).

```python
a = [1, 2, 3]
b = [1, 2, 3]
c = a

print(a == b)   # Output: True  -> same values
print(a is b)   # Output: False -> different objects in memory
print(a is c)   # Output: True  -> c refers to the exact same object as a

print(id(a), id(b), id(c))  # a and c share the same id; b has a different id
```

#### **🎯 Interview Gotcha #1 — Small integer caching (-5 to 256):**
Python pre-caches small integers, so `is` can behave unexpectedly:
```python
x = 100
y = 100
print(x is y)   # Output: True  -> small ints are cached/reused by Python

x = 1000
y = 1000
print(x is y)   # Output: False -> larger ints are NOT cached, separate objects
```

#### **🎯 Interview Gotcha #2 — String interning:**
```python
s1 = "hello"
s2 = "hello"
print(s1 is s2)   # Output: True  -> simple string literals are often interned

s3 = "hello world!"
s4 = "hello world!"
print(s3 is s4)   # Output: False -> longer/complex strings are usually NOT interned
```

**Rule of thumb for interviews:** Always use `==` to compare values. Only use `is` (or `is not`) to check against `None`, e.g. `if x is None:`.

### **5. Membership Operators**

Membership operators test for membership within a sequence, such as a list, string, or tuple. They return `True` or `False` based on whether the value is found in the sequence.

#### **Membership Operators**:
- `in`: Returns `True` if the specified value is found in the sequence.
- `not in`: Returns `True` if the specified value is not found in the sequence.

#### **Examples**:
```python
my_list = [1, 2, 3, 4, 5]
my_string = "Python"

print(3 in my_list)  # Output: True (3 is in the list)
print(6 not in my_list)  # Output: True (6 is not in the list)
print("P" in my_string)  # Output: True ("P" is in the string)
print("z" not in my_string)  # Output: True ("z" is not in the string)
```

#### **🎯 Interview Gotcha — `in` with dictionaries checks KEYS, not values:**
```python
my_dict = {"name": "AJ", "role": "Developer"}

print("name" in my_dict)        # Output: True  -> checks keys
print("AJ" in my_dict)          # Output: False -> "AJ" is a value, not a key
print("AJ" in my_dict.values()) # Output: True  -> explicitly check values instead
```

### **6. Bitwise Operators**

Bitwise operators perform operations on binary representations of integers. These operators are useful for low-level programming tasks like working with bits and bytes.

#### **Common Bitwise Operators**:
- `&`: Bitwise AND (sets each bit to 1 if both bits are 1).
- `|`: Bitwise OR (sets each bit to 1 if one of the bits is 1).
- `^`: Bitwise XOR (sets each bit to 1 if only one of the bits is 1).
- `~`: Bitwise NOT (inverts all the bits).
- `<<`: Left shift (shifts bits to the left by a specified number of positions).
- `>>`: Right shift (shifts bits to the right by a specified number of positions).

#### **Examples**:
```python
a = 5  # In binary: 101
b = 3  # In binary: 011

# Bitwise AND
print(a & b)  # Output: 1 (binary: 001)

# Bitwise OR
print(a | b)  # Output: 7 (binary: 111)

# Bitwise XOR
print(a ^ b)  # Output: 6 (binary: 110)

# Bitwise NOT
print(~a)  # Output: -6 (inverts all bits)

# Left shift
print(a << 1)  # Output: 10 (binary: 1010)

# Right shift
print(a >> 1)  # Output: 2 (binary: 010)
```

#### **🎯 Interview Gotcha — Common bitwise tricks asked in interviews:**
```python
# Check if a number is even or odd using bitwise AND (faster than % in some contexts)
n = 7
print(n & 1)   # Output: 1 -> odd (0 would mean even)

# Swap two numbers without a temporary variable, using XOR
a, b = 5, 9
a = a ^ b
b = a ^ b
a = a ^ b
print(a, b)    # Output: 9 5

# Multiply/divide by powers of 2 using shifts (fast bit tricks)
print(5 << 1)   # Output: 10  -> same as 5 * 2
print(20 >> 2)  # Output: 5   -> same as 20 // 4
```

### **7. Arithmetic Operators**
Python supports basic arithmetic operations like addition, subtraction, multiplication, division, and more.

#### **Common Operators:**
- `+` (Addition)
- `-` (Subtraction)
- `*` (Multiplication)
- `/` (Division)
- `//` (Floor Division)
- `%` (Modulus)
- `**` (Exponentiation)

#### **Examples:**
```python
a = 10
b = 3
print(a + b)  # Output: 13
print(a - b)  # Output: 7
print(a * b)  # Output: 30
print(a / b)  # Output: 3.3333...
print(a // b)  # Output: 3 (Floor Division)
print(a % b)  # Output: 1 (Modulus)
print(a ** b)  # Output: 1000 (Exponentiation)
```

#### **🎯 Interview Gotcha — Floor division and modulus with negative numbers:**
```python
print(-7 // 2)   # Output: -4  -> NOT -3! Floor division rounds toward negative infinity
print(-7 % 2)    # Output: 1   -> Python's % always returns a result with the SAME SIGN as the divisor

print(7 // -2)   # Output: -4
print(7 % -2)    # Output: -1

# Operator precedence: ** binds tighter than unary minus
print(-2 ** 2)   # Output: -4  -> evaluated as -(2 ** 2), NOT (-2) ** 2
print((-2) ** 2) # Output: 4
```
